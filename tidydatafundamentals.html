What is Tidy Data?

Tidy data is a standard way of mapping the structure of a dataset to its meaning. A dataset is "tidy" if its physical structure matches its logical meaning.

The Three Core Rules

1

Variables in Columns

Every attribute (e.g., age, weight) gets its own column.

2

Observations in Rows

Every single event or measurement gets its own row.

3

One Unit per Table

Don't mix levels (like Store Info and Customer Data).

Common "Messy" Patterns

Spot these common issues to know when your data needs tidying:

1. Headers are Values

Instead of a "Week" column, you have columns named "Week 1", "Week 2".

Bad: [ID, Week1, Week2]

2. Multiple Variables in One

Two measurements crammed into one cell (e.g., Height/Weight).

Bad: [User, H_W] -> [John, "170/65"]

3. Variable Split Apart

One variable is broken into pieces that should be together (e.g., Date parts).

Bad: [Day, Month, Year]

4. Mixed Units

Individual data mixed with summary data in the same list.

Bad: [Student_Score] + [Class_Average_Row]
