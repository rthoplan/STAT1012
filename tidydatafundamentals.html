What is Tidy Data?

Tidy data is a standard way of mapping the structure of a dataset to its meaning. A dataset is "tidy" if its physical structure matches its logical structure, making it easy for computers to process and for humans to read.

The Three Core Rules

1

Each variable forms a column.

A variable is an attribute you are measuring (like "Height," "Temperature," or "Date"). Each one gets its own dedicated column.

2

Each observation forms a row.

An observation is a single "event" or measurement. If you measure one person twice, that person should have two separate rows.

3

Each type of observational unit forms a table.

Keep different levels of data separate. Don't mix data about a "Store" with data about a "Customer Purchase" in the same table.

Example: Temperature Records

Untidy Format

City Morning_Temp Evening_Temp
London 15°C 12°C

The Problem: The variable "Time" is hidden in the column names. The data (Morning/Evening) is acting as a header.

Tidy Format

City Time Temp
LondonMorning15°C
LondonEvening12°C

The Fix: "Time" and "Temp" are now distinct variables. Each row represents a single measurement.
